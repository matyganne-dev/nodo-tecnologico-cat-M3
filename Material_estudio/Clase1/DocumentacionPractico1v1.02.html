1. Metodología Scrum aplicada a la educación

1.1 Introducción a Scrum

Scrum es una metodología ágil diseñada para organizar y gestionar el desarrollo de proyectos complejos. Su enfoque permite a los equipos trabajar de forma colaborativa y eficiente para alcanzar objetivos comunes mediante ciclos cortos de trabajo denominados sprints.

Scrum es un marco de trabajo ágil orientado a la mejora continua y a la entrega incremental de productos funcionales. Se estructura en ceremonias, roles y artefactos que garantizan un flujo de trabajo organizado, transparente y colaborativo.

En el ámbito educativo, Scrum puede adaptarse como estrategia pedagógica para fomentar el trabajo en equipo, la responsabilidad compartida, la planificación organizada y la retroalimentación constante.

1.2 Ceremonias clave de Scrum

Scrum se organiza en torno a cuatro ceremonias fundamentales que aseguran el progreso continuo y la mejora del equipo en cada sprint:

1. Sprint Planning (Planificación del Sprint)

  • Cuándo: Al inicio de cada sprint (generalmente los lunes).
  • Qué es: Reunión donde se define el trabajo que se realizará durante el sprint.
  • Objetivo: Establecer un plan claro sobre las tareas a desarrollar y asignar responsabilidades dentro del equipo.

2. Daily Stand-up (Reunión diaria)

  • Cuándo: Internamente todos los días. Reuniones formales con el docente los lunes, miércoles y viernes.
  • Duración: 5 a 10 minutos (puede extenderse si es necesario).
  • Dinámica: Cada integrante responde tres preguntas:
  1. ¿Qué hice ayer?
  2. ¿Qué haré hoy?
  3. ¿Tengo algún impedimento o bloqueo?

El docente utiliza este espacio como instancia de seguimiento y resolución de problemas. Se atenderán las dudas previamente cargadas en el formulario de consultas.

  • Objetivo: Mantener alineado al equipo y detectar obstáculos de manera temprana.

3. Sprint Review (Revisión del Sprint)

  • Cuándo: A partir del segundo sprint. Se realiza luego de las Daily con personas seleccionadas.
  • Qué es: Presentación del trabajo completado en el sprint anterior.
  • Duración: 5 a 15 minutos (puede extenderse).
  • Objetivo: Validar el trabajo realizado y recibir retroalimentación del docente para mejorar el producto y el proceso.

4. Sprint Retrospective (Retrospectiva)

  • Cuándo: Luego de la Sprint Review.
  • Qué es: Espacio de reflexión interna del equipo.
  • Preguntas clave:
  • ¿Qué salió bien?
  • ¿Qué se puede mejorar?
  • Objetivo: Identificar oportunidades de mejora para el siguiente sprint.

1.3 Roles en Scrum

En este módulo, los equipos estarán organizados en grupos de cuatro personas, con roles definidos:

Product Owner / Stakeholder

Este rol será ejercido por el docente.

  • Define los requerimientos.
  • Prioriza tareas.
  • Evalúa si el trabajo cumple con los criterios solicitados.

Desarrolladores

  • Ejecutan las tareas asignadas.
  • Participan activamente en las ceremonias.
  • Colaboran entre sí para cumplir los objetivos del sprint.

1.4 Uso de plataformas para la gestión ágil de proyectos

Cada equipo contará con acceso individual a una herramienta de gestión de proyectos para organizar sus tareas prácticas, visualizar el backlog y dar seguimiento al progreso del sprint.

2. Introducción a Node.js

2.1 ¿Qué es Node.js?

Node.js es un entorno de ejecución de JavaScript que permite ejecutar código del lado del servidor, más allá del navegador. Está construido sobre el motor V8 de Google Chrome y se caracteriza por su eficiencia y escalabilidad gracias a su arquitectura basada en eventos y su modelo de entrada/salida no bloqueante.

Node.js permite desarrollar aplicaciones capaces de manejar múltiples solicitudes simultáneas sin necesidad de crear múltiples hilos, lo que lo convierte en una excelente opción para aplicaciones en tiempo real.

2.2 Arquitectura de Node.js

Node.js utiliza:

  • Modelo de un solo hilo (single-threaded).
  • Bucle de eventos (event loop).
  • Operaciones asíncronas no bloqueantes.

Este enfoque es especialmente eficiente en aplicaciones con gran cantidad de operaciones de entrada/salida, como servidores web y APIs.

2.3 Instalación de Node.js y configuración del entorno

Instalación de Node.js

  1. Descargar la versión LTS desde: https://nodejs.org
  2. Ejecutar el instalador con configuración predeterminada.
  3. Verificar instalación en terminal:

node -v

npm -v

Instalación de Visual Studio Code

  1. Descargar desde: https://code.visualstudio.com
  2. Instalar siguiendo el asistente.
  3. Instalar extensiones recomendadas:
  • Node.js Extension Pack
  • ESLint
  • Prettier

2.4 Casos de uso de Node.js

Node.js es ideal para:

  • Aplicaciones en tiempo real (chats, juegos en línea, apps colaborativas).
  • APIs RESTful.
  • Automatización de tareas mediante scripts.
  • Aplicaciones de streaming de audio y video.

2.5 Ventajas y desventajas

Ventajas

  • Alta escalabilidad.
  • Uso de JavaScript tanto en frontend como backend.
  • Gran ecosistema de paquetes a través de npm.

Desventajas

  • No es ideal para tareas altamente intensivas en CPU.
  • Posible complejidad en el manejo de callbacks (mitigable con Promises y async/await).

2.6 Comparativa con otras tecnologías

Node.js vs Python (Flask/Django)

  • Node.js ofrece mejor rendimiento en operaciones I/O.
  • Python puede resultar más simple para aplicaciones científicas o de análisis de datos.

Node.js vs Java (Spring Boot)

  • Node.js utiliza modelo de un solo hilo con event loop.
  • Java utiliza múltiples hilos para la concurrencia.
  • Java posee un ecosistema más maduro para aplicaciones empresariales tradicionales.
  • Node.js permite desarrollos más rápidos y ligeros.

2.7 Ejemplos prácticos

2.7.1 Servidor básico en Node.js

Objetivo: Crear un servidor que responda con "Hola Mundo" a solicitudes HTTP.

import http from 'http';

const hostname = '127.0.0.1';

const port = 3000;

const server = http.createServer((req, res) => {

  res.statusCode = 200;

  res.setHeader('Content-Type', 'text/plain');

  res.end('Hola Mundo\n');

});

server.listen(port, hostname, () => {

  console.log(`Servidor corriendo en http://${hostname}:${port}/`);

});
